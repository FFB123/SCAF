每个方法的技术细节

Baseline（随机/基础提示词）

目标
用固定基础提示词直接完成四标签分类，作为最小基线。

核心流程
1. 从 `PROMPT_TEMPLATES['baseline']` 构造全量 `prompts`。
2. 调用 `vllm_batch_inference(llm, sampling_params, prompts, batch_size=256)`。
3. 用 `parse_response` 解析输出。
4. 保存 `baseline_responses` / `baseline_responses_sft`，并评估。

关键参数
- `temperature=0.7`
- `top_p=0.8`
- `max_tokens=100`
- 停止词包含 `"<|endoftext|>", "<|im_end|>"` 及多种中文模式。

关键实现点
- 批推理由 `utils.py` 的 `vllm_batch_inference` 完成。
- 输出先经过 `truncate_repetitive_content`，减少重复生成污染。



APE（Automatic Prompt Engineering）

APE 分两段：

A 段：提示词优化（调用 DeepSeek API）

1. `call_deepseek_api()` 直连 `https://api.deepseek.com/v1/chat/completions`。
2. `generate_prompt_candidates()` 生成候选提示词。
3. `filter_candidates()` 清洗候选。
4. `evaluate_prompt()` 在 dev 集逐样本调用 API 打分（macro-F1）。
5. `select_best_prompt()` 选最优，保存 `best_prompt.json`。
B 段：正式评测推理（本地 vLLM）
读取 best_prompt.json。
build_ape_prompt_with_descriptions 拼接“标签定义+示例+输出协议”。
调 vllm_batch_inference(..., batch_size=256) 批量推理。

关键实现点
APE 自动优化的过程是 API （DeepSeek）驱动。
最终测试集推理是本地 Qwen(vLLM)。




Auto-CoT（自动思维链）


Auto-CoT 的推理链链生成来自 API（DeepSeek）。
Auto-CoT 的分类推理来自本地 Qwen(vLLM)。


阶段A：CoT 示例生成 （DeepSeek API）
select_representative_samples 从训练集按标签挑代表样本。
generate_cot 调 DeepSeek API 生成“思考 + 输出”。
format_cot_examples 汇总写入 results/cot_examples.txt。
阶段B：Auto-CoT 推理（本地 vLLM）
读取 `cot_examples.txt` 拼接 Auto-CoT prompt。
使用 PROMPT_TEMPLATES['autocot']（内含 few-shot 思维链示例）。
调 vllm_batch_inference_autocot，其内部创建新采样参数：
max_tokens=1024（比 baseline 的 100 更长，利于推理）
temperature/top_p/stop 继承基础设置。
实际生成仍是 llm.generate(...)，只是参数更偏向长推理文本。
解析用 parse_response（同 parse_ape_response）。
评估用 evaluate_results。
缓存
autocot_responses（SFT版：autocot_responses_sft）。
方法本质
通过“示例化推理轨迹”提升模型对复杂标签的判别一致性，代价是显著增加生成长度和时延。



Self-Consistency（自洽性投票）

目标
对同一输入采样多次，再投票融合，降低单次随机采样误差。

流程：
用 baseline 模板构造 prompt
对每条样本生成 k=5 次
每次用 temperature=0.9（多样性更强）, top_p=0.8, max_tokens=100
truncate_repetitive_content 清理重复
每条样本把 5 次结果解析成结构化标签
vote_for_answer 投票得到最终标签
评估并打印 Acc/Prec/Rec/F1

核心流程
1. 设定 `K_SELF_CONSISTENCY`（本实验设置为5， 考虑到训练时长）。
2. 对每个样本 prompt，循环 K 次：
   - `outputs = llm.generate([prompt], sc_sampling_params)`
   - `response = truncate_repetitive_content(...)`
3. 对 K 份响应逐个 `parse_response`。
4. 调 `vote_for_answer(k_predictions, K)`：
   - `toxic / toxic_type / expression` 做多数投票。
   - `target` 用“票数 >= K/2”保留。
5. 记录 `consistency_scores` 并保存。
